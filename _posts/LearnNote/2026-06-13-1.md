<h1>第一部分：問題描述</h1>

<h2>Step 1：問題描述</h2>
當開啟 ComfyUI 容器時，突然無法啟動，問題是昨天之前都是正常的
<br/>目前容器給出的關鍵錯誤是 : 

``` Markdown
RuntimeError: The NVIDIA driver on your system is too old (found version 12090)
```

<br/>實際應用上 : 容器應保持版本不動 -> 容器版本不動，不換顯卡就不應出現顯本版本問題 
<br/>因此容器啟動方式需調整徹底根除 **mmartial/comfyui-nvidia-docker:ubuntu24_cuda12.8-latest**

<h1>第三部分：徹底根除方法</h1>

<h2>Step 1：保持版本穩定 - 前言</h2>
應依照自己的使用情況，若在開發研究的環境中，需要時時使用最新的 ComfyUI + 顯卡驅動(同步更新可以確保使用到更優化、效能更好)
<br/>另一種徹底根除更新驅動、ComfyUI 的方案 : **移除 lastest 並且，並且固定下穩定版本**
<br/><img src="/assets/image/LearnNote/2026_06_13/001.png" alt="" width="50%" height="50%" />
<br/><br/>

<h2>Step 2：找到 ComfyUI Image 穩定版本</h2>
進入 <a href="https://hub.docker.com/r/mmartial/comfyui-nvidia-docker/tags">DockerHub</a> 找尋帶有版本號的腳本
<br/>以此圖為例，我們 Copy 了此腳本，版本號 20260509，而非 latest

``` shell
docker pull mmartial/comfyui-nvidia-docker:ubuntu24_cuda13.2-20260509
```

<br/><img src="/assets/image/LearnNote/2026_06_13/007.png" alt="" width="50%" height="50%" />
<br/><br/>

<h2>Step 3：建立 ComfyUI 容器使用指定版本號</h2>
然後重新建立 ComfyUI 容器使用此腳本，未來只要顯卡不更換(更新)，那麼 ComfyUI 將能避免此問題再發生

<br/>對應此篇文章這個步驟： <a href="https://gotoa1234.github.io/2026/04/25/1.html#h2_24">[第四部分：ComfyUI 容器啟動] - Step 2：安裝 ComfyUI - 執行安裝</a>

``` shell
cd ~/comfyui && docker run -it \
  --runtime nvidia \
  --gpus all \
  -v "$(pwd)/run:/comfy/mnt" \
  -v "$(pwd)/basedir:/basedir" \
  -v /usr/lib/wsl:/usr/lib/wsl \
  -e LD_LIBRARY_PATH=/usr/lib/wsl/lib \
  -e WANTED_UID=$(id -u) \
  -e WANTED_GID=$(id -g) \
  -e BASE_DIRECTORY=/basedir \
  -e SECURITY_LEVEL=normal \
  -p 127.0.0.1:8188:8188 \
  --name comfyui-nvidia \
  --rm \
  mmartial/comfyui-nvidia-docker:ubuntu24_cuda13.2-20260509
```
