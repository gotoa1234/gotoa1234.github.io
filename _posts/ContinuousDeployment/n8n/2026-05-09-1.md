<h1>第一部分：問題描述 & 狀況</h1>

<h2>Step 1：問題描述</h2>
忘記密碼或 Cookie 已經遺失，訪問 n8n 造成無法登入系統
<br/> <img src="/assets/image/ContinuousDeployment/n8n/2026_05_09/001.png" width="50%" height="50%" />
<br/><br/>

<h2>Step 2：重設密碼 - 自架出現的狀況</h2>
下方有按鈕 **Forgot my password** 一般網站點擊後，都可以收到一封重設密碼的 Email，但我們是透過自架設，通常是不會額外配置 SMTP (自己用)
<br/>因此點擊後，出現以下訊息：

``` Markdown
Please contact your admin. n8n isn't set up to send email right now.
```

<br/> <img src="/assets/image/ContinuousDeployment/n8n/2026_05_09/001_2.png" width="50%" height="50%" />
<br/><br/>


<h2>Step 3：補充資訊配置 SMTP - 可略過</h2>
若想要設置容器 SMTP 可以參考此篇 <a href="https://gotoa1234.github.io//2025/12/27/1.html#h1_0">0024. 【NAS 進階】Synology 免費私人知識庫 BookStack 進階設定：Gmail 郵件通知與 Webhook 自動化整合測試教學</a>
<br/>要透過對 Docker-Compose.yml 配置 SMTP 讓容器可以發送 Email
<br/><br/>




<br/><br/>
<h1>第二部分：解決方案</h1>

<h2>Step 1：解決方案</h2>
最快的解決方案：

``` Markdown
略過 Email，直接重置
```

<br/><br/>

<h2>Step 2：登入 Ubuntu 主機</h2>
登入搭建 n8n 的 Ubuntu 主機
<br/> <img src="/assets/image/ContinuousDeployment/n8n/2026_05_09/002.png" width="50%" height="50%" />
<br/><br/>


<h2>Step 3：找出 n8n Container</h2>
輸入以下指令，找出 Container ，紀錄容器的 **ID 或 Name**

``` shell
docker ps -a
```

<br/> <img src="/assets/image/ContinuousDeployment/n8n/2026_05_09/003.png" width="100%" height="100%" />
<br/><br/>



<h2>Step 4：找出 n8n Container</h2>
輸入以下指令，進入容器執行 reset
<br/>container_name_or_id : 替換成 ID 或 Name

``` shell
docker exec -it <container_name_or_id> n8n user-management:reset
```

<br/> <img src="/assets/image/ContinuousDeployment/n8n/2026_05_09/004.png" width="50%" height="50%" />
<br/><br/>

<h2>Step 5：重新訪問 n8n WebUI</h2>
再次訪問 n8n 容器

``` markdown
http://{您的IP}:5678/home/workflows
```

<br/>會跳出重新設定密碼的選項：
<br/> <img src="/assets/image/ContinuousDeployment/n8n/2026_05_09/005.png" width="50%" height="50%" />
<br/><br/>


<h2>Step 6：重設完成</h2>
登入後，會發現資料仍會保留，可正常操作 n8n 系統
<br/> <img src="/assets/image/ContinuousDeployment/n8n/2026_05_09/006.png" width="100%" height="100%" />
